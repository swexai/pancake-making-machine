# STM32F446RE IOC Setup Summary

## What Was Created

This folder contains a complete template for the Pancake-Making Machine STM32F446RE firmware, with three components:

### 1. **Configuration Documentation** (`IOC_CONFIGURATION_GUIDE.md`)
   - **Pin mapping** for all subsystems (thermal, motion, pump, safety, HMI)
   - **Peripheral details** (SPI1, TIM1, TIM2, TIM4, USART2, GPIO)
   - **NVIC priority assignments** (Safety @ 1 kHz is highest)
   - **Step-by-step instructions** for STM32CubeMX project setup
   - **FreeRTOS configuration** (task stack sizes, heap)
   - **Post-generation checklist**

### 2. **Project Template with HAL Configuration**
   Files in `config/`:
   - `stm32f4xx_hal_conf.h` — HAL module enables/disables
   - `FreeRTOSConfig.h` — Kernel, heap, task priorities
   - `pancake-making-machine.ioc` — [Generated by STM32CubeMX]

### 3. **Application Skeleton** (`src/`)
   Headers for 5 FreeRTOS tasks:
   - `thermal_control.h` — PT100 + MAX31865 + SSR (25 Hz sample, 10 Hz SSR drive)
   - `motion_control.h` — NEMA23 stepper + TB6600 (1 kHz STEP)
   - `pump_control.h` — Dispense pump PWM (10 Hz)
   - `safety.h` — E-STOP + cover switch (1 kHz monitor)
   - `hmi_logger.h` — Console I/O (1 Hz logging)
   - `main.c` — Entry point with task creation

### 4. **Build System**
   - `CMakeLists.txt` — CMake-based build (ARM GCC toolchain)
   - `README.md` — Quick-start guide & workflow

## Next Steps: Generate & Integrate

### Step 1: Create STM32CubeMX Project
1. Open STM32CubeMX
2. **Create New Project** → Choose **STM32F446RETx**
3. Follow the **IOC_CONFIGURATION_GUIDE.md** (sections 3–7) to configure:
   - System clock (180 MHz HSE)
   - SPI1, timers, USART2, GPIO
   - FreeRTOS middleware
4. **Project → Generate Code** → Output to `firmware/stm32f446re/`

### Step 2: Integrate Generated Code
- Copy HAL files (`*.c`, `*.h`) from `Drivers/STM32F4xx_HAL_Driver/Src` → `hal/`
- Copy startup & system files → `hal/`
- Copy `stm32f4xx_it.c` (interrupt handlers) → `hal/`
- Review `main.c` generated by CubeMX, merge with skeleton in `src/main.c`

### Step 3: Implement Task Stubs
Replace the header declarations with full implementations:
- `thermal_control.c` — SPI read loop, PID algorithm
- `motion_control.c` — STEP ISR, direction/speed control
- `pump_control.c` — PWM duty cycle management
- `safety.c` — GPIO polling, fault state machine
- `hmi_logger.c` — UART command parsing, printf logging

### Step 4: Build & Flash
```bash
cd firmware/stm32f446re/
mkdir build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release
make
# Output: pancake-making-machine.elf, .bin, .hex
```

Flash using STM32CubeIDE, ST-Link utility, or OpenOCD.

## Key Design Decisions

### Clock & Timing
- **System Clock**: 180 MHz (HSE 8 MHz × 45 ÷ 2)
- **APB1**: 45 MHz (prescaler /4) — for SPI, TIM2, TIM4
- **APB2**: 90 MHz (prescaler /2) — for USART2, TIM1
- **SysTick**: 1 kHz FreeRTOS tick

### Task Scheduling
| Task | Rate | Priority | Purpose |
|------|------|----------|---------|
| Safety | 1 kHz | 5 | Hardware safety chain monitor |
| ThermalCtrl | 25 Hz | 4 | PID loop (SSR driven @ 10 Hz) |
| MotionCtrl | Timer-driven | 3 | Stepper STEP generation (~1 kHz) |
| PumpCtrl | 10 Hz | 2 | Dispense PWM control |
| HMI | 1 Hz | 1 | Console logging, command parsing |

### Thermal Control
- **Temperature sensor**: PT100 RTD read via MAX31865 over SPI1
- **Heater drive**: SSR PWM on TIM1_CH1 @ 10 Hz (~50% base duty)
- **PID loop**: 25 Hz sampling, tuned for ±3 °C stability @ 210–230 °C

### Motion Control
- **Motor**: NEMA23 with TB6600 driver
- **Control**: STEP/DIR/EN on GPIO (PB3, PB4, PB5)
- **Frequency**: ~500 Hz STEP rate = 0.2 rev/s rotation
- **Timing**: TIM2 toggle mode @ 1 kHz → 500 Hz STEP output
- **Homing**: GPIO limit switch (not shown in current pin map — add if needed)

### Dispense
- **Pump**: Variable duty PWM on TIM4_CH1 @ 10 Hz
- **Flow**: Target 28.3 mL/s (141 mL total over 5 s dispense cycle)
- **Manifold**: 7-way split (outer rings get higher mL/s per equal-length branches)

### Safety
- **E-STOP**: Dual-channel hardwired cutoff of SSR & θ-EN (PA0, PA1 GPIO inputs + hardware chain)
- **Cover Switch**: NC (normally closed) — open = de-energize (PA1)
- **Software**: Safety_Task monitors and enforces firmware-level shutdown

## Testing Checklist

After build & flash:
- [ ] **UART Console**: Connect serial monitor (115200 baud) → see "System startup" message
- [ ] **Thermal**: Use multimeter on PT100 → verify SPI read ≈ room temp
- [ ] **Motion**: GPIO scope on PB3 (STEP) → verify ~500 Hz toggle
- [ ] **Pump**: GPIO scope on PB6 (PWM) → verify 10 Hz @ variable duty
- [ ] **Safety**: Short PA0 (E-STOP) → motor & heater de-energize (hardware + firmware)
- [ ] **HMI**: Type command via serial → motor homes, heater targets 210 °C

## Troubleshooting

### Build Errors
- **Missing `startup_stm32f446xx.s`**: Generated by CubeMX; ensure it's in `hal/`
- **Undefined HAL symbols**: Run CubeMX code generation with "Copy all used library files into project" ✓

### Flash Failures
- Ensure **ST-Link is connected** (USB → dev board)
- Check STM32CubeIDE → **Window → Show View → Debug → Debugging as ST-Link (SWD)**

### Runtime Issues
- **Stack overflow**: Increase `configMINIMAL_STACK_SIZE` in `FreeRTOSConfig.h`
- **SPI not communicating**: Verify SPI clock (90 MHz / 16 prescaler = 5.625 MHz for MAX31865)
- **PWM frequencies wrong**: Check timer prescaler & ARR values in IOC guide Section 3
